Question:-

How does “InterProcessSignalPropagator” works?

 

Answer:-

Followings are the major steps are followed while an signal is propagated over TCP/IP

1.      A “local outgoing signal” of a “local object” is emmited

2.      This signal is converted into a “MetaCall” event and posted to local “InterProcessSignalPropogator”

3.      Local “InterProcessSignalPropogator” sends this “MetaCall” event over TCP/IP to remote process’s “InterProcessSignalPropogator”

4.      Remote process’s “InterProcessSignalPropogator” receives the data and creates “MetaCall” event

5.      Using the “placeMetaCall” event the “incoming signal” is emmited

6.      The “local slot” connected to this “incoming signal” is called
