Question:-

How to test/run “InterProcessSignalPropagator” example?

 

Answer:-

To test/run “InterProcessSignalPropagator” example following are the prerequisites:-

1.      SVN client and internet connectivity to download the source code

2.      Qt development environment, versioned Qt 4.3.3 or greater

 

To test/run “InterProcessSignalPropagator” example following steps should be done:-

1.      Startup a Qt development environment shell prompt

2.      Download the source using following command

   svn checkout http://interprocesssignalpropagator.googlecode.com/svn/trunk/ interprocesssignalpropagator-read-only

3.      Change the working directory to “InterProcessSignalPropagator”

4.      To generate the makefiles run “qmake” command

5.      To build the project run “make” or “nmake” or “gmake” command

6.      Change the working directory to “InterProcessSignalPropagator/Examples/remotecontrolledcar/car”

7.      Run the “car” executable

8.      Startup another Qt development environment shell prompt

9.      Change the working directory to “InterProcessSignalPropagator/Examples/remotecontrolledcar/controller”

10. Run the “controller” executable

11. Now you can play the “remote controlled car” game!!

 

Note:-

To test/run the example on different machines you need convey following things to both the application:-

1.      The IP address of the machine where “car” is running

2.      The port number on which “car” is listening

 

Sample command line could be

car “192.168.20.100” “5050”

controller “192.168.20.100” “5050”
